PocketC Architect Desktop is a Windows-based editor and compiler for the OrbC language.

Building a Project

Selecting Project | Build Project compiles the current project file using the settings in the Compile dialog and the properties in the @app resource. The first source file opened is considered to be the project file. To explicitly change the project file, select Project | Set Project File or right-click within a file and select Set as Project.

Compile Dialog

The compile dialog gives you two options:

  • Generate standalone application - Selecting this check box will create a standalone application - a .prc file which contains an embedded runtime and any required native add-ins. When developing an application, it is generally better not to create standalone applications, because using the debug version of OrbFormsRT-debug.prc will enable you to find bugs in your code more easily. When distributing an application, it is best to use standalone applications so that your users will only need to install one file. Legacy PocketC native libraries are not embedded in a standalone application. The demo version is not able to build standalone applications, so this checkbox will be disabled.
  • Enable debug code- Selecting this checkbox enables all the features provided by the debug support.

File Extensions

PocketC Architect uses three file extensions. The compiler and editor treats all files types the same, but their intent is as follows:

  • OrbC Project File (.ocp) - An OrbC source file which is the primary project file. If other files are used in the project, they are included from this file. Generally the application resources are also defined in this file.
  • OrbC Source File (.oc) - An OrbC source file other than the primary project file. It is convenient to use a separate .oc file for each form and gadget implementation in your application.
  • PocketC Source File (.pc) - A legacy PocketC source file.

QuickDocs

When you place the cursor on a function name, it displays the function declaration along with short documentation. The documentation for built-in functions is provided by the compiler. However, you can add documentation to your own functions very easily by adding an @doc directive before the function definition. For example:

@doc "Create a new button";
addButton(int x, int y, string name) { ... }

Uploading to the Emulator or Simulator

The application's .prc will be automatically installed on the emulator or simulator if it is running. If the .prc file on the device is in use (i.e. the application is running), the file will not upload successfully. Select Options | Upload on Build to enable or disable this feature.

Automatic Updates

PocketC Architect can automatically look for updates each time it is run. Select Options | Auto Update to enable or disable this feature. Checking for updates involves downloading a version info file from the OrbWorks server. No personally identifiable information is transfered.

Compiling from the Command Line

To compile a project from the command line, you can use the included command line compiler, oc.exe.

oc.exe -pocketc [-standalone] [-debug] project.ocp

The -pocketc switch must always be used to provide PocketC compatibility. The remaining switches relate directly to the options specified in the Compile dialog when building from the editor.
